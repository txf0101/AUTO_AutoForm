# AutoForm Agent 项目协作约束

## 表述规范

- 避免使用先否定再肯定的对照句式。表达差异时，直接说明适用范围、依据和结论。
- 减少由多个短词连续用连字符拼接形成的表述。确有命令、参数、文件名、包名、模型名或外部原文需要保留时，按原样书写。
- 命令、路径、版本、软件行为、官方能力和技术结论都要给出依据。依据可以来自本机文件、源码行、注册表、官方文档、命令输出或用户提供的材料。
- 未经验证的判断要标明为待验证或推测，并说明下一步验证方法。
- 论文、报告或正式说明类输出采用稳健、连续、克制的学术表达，语言风格参考 2015 至 2020 年博士论文常见写法。

## 项目资料来源要求

- AutoForm 官方能力优先依据本机安装目录、`C:\ProgramData\AutoForm`、Windows 注册表、开始菜单快捷方式、随安装包提供的脚本和头文件。
- Agent 能力优先依据当前工作区源码、测试、命令帮助输出和实际执行记录。
- 如果资料来源不足，只记录已经确认的事实，并把需要补充验证的内容列入后续工作。

## 检查要求

- 修改文档或生成资料后，至少进行一次内容检查。
- 修改代码后，运行与变更范围匹配的测试或语法检查。
- 对 Word、PDF、表格等交付物，优先进行结构读取或渲染检查；缺少渲染工具时，在交付说明中说明采用的替代检查方式。

## 新手文档维护

- 面向零代码基础用户的中文上手文档为 `docs/beginner_onboarding_zh.md`。
- 后续修改安装环境、启动脚本、CLI 命令、MCP 入口、前端入口、目录结构、测试命令或 README 中的新手相关说明时，必须同步检查并按需要更新该文档。
- 如果变更后确认该文档无需调整，应在交付说明、提交说明或变更记录中写明已经检查过该文档。
