# 02 启动运行与界面测试

## 启动前先确认

维护者接手后先确认三件事：

1. 当前目录是项目根目录。
2. Python 环境能导入 `autoform_agent`。
3. 本地端口没有被旧进程占用。

常用项目根目录：

```powershell
F:\【项目和任务】\EIT\2026\AUTO_AutoForm
```

常用 Python 环境：

```powershell
C:\Users\Tang Xufeng\.conda\envs\afagent\python.exe
```

## 启动器方式

推荐先用启动器。启动器负责检查环境、启动 HTTP bridge、启动静态前端服务，并把日志写到 `output/launcher_logs`。

```powershell
.\start_autoform_agent.ps1
```

如果你改了后端或前端，怀疑页面仍在读旧进程，可以使用重启参数：

```powershell
.\start_autoform_agent.ps1 -RestartServices
```

## 手动启动方式

如果启动器异常，可以手动启动两个服务。

第一步，启动 HTTP bridge：

```powershell
& 'C:\Users\Tang Xufeng\.conda\envs\afagent\python.exe' -m autoform_agent.http_bridge --host 127.0.0.1 --port 4317
```

第二步，从项目根目录启动静态前端服务。静态服务的根目录必须是项目根目录，因为页面要读取 `fixtures/run_events_demo.jsonl`。

```powershell
& 'C:\Users\Tang Xufeng\.conda\envs\afagent\python.exe' -m http.server 8765 --bind 127.0.0.1
```

第三步，打开页面：

```text
http://127.0.0.1:8765/apps/workbench/index.html?bridge=http
```

## 健康检查

HTTP bridge 健康检查：

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:4317/health'
```

前端静态文件检查：

```powershell
Invoke-WebRequest -UseBasicParsing -Uri 'http://127.0.0.1:8765/apps/workbench/index.html?bridge=http'
```

页面打开后应能看到：

- 左侧用户输入区域。
- 中间状态总结。
- 中间下方 Agent 图谱。
- 右侧命令输出。
- 下方凭据边界和 token 面板。

## Agent 图谱界面验收

当前图谱必须固定显示九个业务节点：

1. 中心Agent
2. 需求与工艺规划Agent
3. 几何与数据Agent
4. 材料Agent
5. 工艺设置Agent
6. 求解执行Agent
7. 后处理Agent
8. 诊断与优化Agent
9. 报告整理Agent

验收标准：

- 初始状态全部为灰白待命。
- 单步回放运行事件时，对应节点变成绿色工作态。
- 回放结束后，不应有绿色运行态残留。
- User、UI、Runtime、Gateway 这类内部调试节点不应显示成图谱方块。

## 页面按钮测试

进入页面后，可以按这个顺序测试：

1. 点击“重置”，让 fixture 回到初始状态。
2. 点击“单步”，观察命令输出和 Agent 图谱。
3. 多点几次“单步”，看到某个 Agent 进入工作态。
4. 点击“跑完”，观察图谱回到灰白待命。
5. 在 prompt 框输入请求，点击“发送”，观察 `/api/agent` 是否返回运行结果。

## 本机执行选项测试

页面有“允许本机执行示例工程”选项。这个选项很关键：

- 不勾选时，页面只发送用户请求和配置，后端可以解析意图，但复制、打开窗口、求解等动作会被网关阻断。
- 勾选时，页面会把本机执行批准状态发给后端。后端仍然负责判断工具、参数和风险。

维护者测试 `AutoComp_R13` 复制与打开窗口链路时，先看后端返回的工具状态：

- `autoform_resolve_project` 应先完成，返回官方示例路径。
- `autoform_project_run` 在未批准时应返回 `blocked_requires_approval`。
- 已批准时，`copy_project` 和 `open_gui` 才可以进入受控动作。

若需要保存一张页面截图，可以使用本机 Chrome 的无头截图模式：

