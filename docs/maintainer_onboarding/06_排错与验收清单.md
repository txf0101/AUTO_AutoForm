# 06 排错与验收清单

本清单用于开发完成前的最后确认，也用于定位常见问题。

## 页面打不开

先查两个地址：

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:4317/health'
Invoke-WebRequest -UseBasicParsing -Uri 'http://127.0.0.1:8765/apps/workbench/index.html?bridge=http'
```

判断：

- bridge 不通：看 `output/launcher_logs`，再重启 `start_autoform_agent.ps1 -RestartServices`。
- 前端不通：确认静态服务是否从项目根目录启动。
- 页面像旧版本：重启服务，清理浏览器缓存或换无头截图核验。

## Agent 图谱不对

检查文件：

- `apps/workbench/app.js`
- `apps/workbench/styles.css`
- `apps/workbench/tests/smoke_test.py`
- `apps/workbench/tests/smoke-test.mjs`

验收：

- 只有九个业务 Agent。
- 内部 role_id 通过 `AGENT_NODE_ALIASES` 映射。
- 运行态是绿色。
- 完成态是灰白待命。
- 回放结束后没有绿色残留。

## 中心 Agent 没有调用工具

检查顺序：

1. `agent_runtime.py` 是否识别了 prompt。
2. `_payload_agent_tool_requests()` 是否生成工具请求。
3. `build_center_agent_plan()` 是否带入 `tool_requests`。
4. `AgentToolGateway.call_tool()` 是否返回阻断或完成。
5. 前端是否渲染 `tool_requested`、`tool_completed` 或 `tool_blocked`。

未批准时返回 `blocked_requires_approval` 是预期结果。它说明网关工作正常。

## AutoComp_R13 找不到

检查：

- `autoform_core/paths.py` 是否能发现安装。
- `autoform_core/inventory.py` 是否能列出官方示例。
- `resolve_project_input(example_name="AutoComp_R13")` 是否返回官方示例路径。
- 环境变量 `AUTOFORM_TEST_DIR` 是否覆盖了测试目录。

常用命令：

```powershell
& 'C:\Users\Tang Xufeng\.conda\envs\afagent\python.exe' -m autoform_agent.cli examples
```

## 复制、打开窗口、求解混在一起

先看 `project_workflow.py` 的三个参数：

| 参数 | 含义 |
| --- | --- |
| `copy_project` | 是否复制 `.afd` 到运行目录。 |
| `open_gui` | 是否打开 AutoForm Forming 窗口。 |
| `execute` | 是否提交求解器执行。 |

正确关系：

- 只复制：`copy_project=True`、`open_gui=False`、`execute=False`。
- 复制并打开窗口：`copy_project=True`、`open_gui=True`、`execute=False`。
- 求解：`execute=True`，通常也会复制工程。

打开窗口不应自动提交求解。

## API key 泄露风险

检查：

- 页面只展示 key 来源、配置状态和短指纹。
- 后端响应中不出现明文 key。
- 日志、复盘和截图不包含明文 key。
- 测试中使用的 key 必须是伪值。

常用搜索：

```powershell
rg -n "sk-|api_key|API key|DeepSeek_V4_API" README.md docs apps/workbench autoform_agent tests
```

看到变量名不代表泄露；看到真实 key 值才是问题。

## pytest 临时目录权限问题

如果出现 `PermissionError`，并且堆栈指向 pytest 临时目录创建，可以指定固定临时目录：

```powershell
New-Item -ItemType Directory -Force -Path tmp\pytest_direct | Out-Null
& 'C:\Users\Tang Xufeng\.conda\envs\afagent\python.exe' -m pytest -q --basetemp=tmp\pytest_direct
```

如果指定后仍失败，检查该目录是否被其他进程占用。

## Computer Use 不可用

Computer Use 依赖本机原生管道。若初始化报 `native pipe is unavailable` 或系统找不到指定管道，记录阻塞并停止 Computer Use 路线。可替代检查包括：

- HTTP bridge 请求。
- 浏览器 DOM 读取。
- 无头 Chrome 截图。
- Python 测试。
- Node 前端烟雾测试。

最终说明中要写清楚 Computer Use 的真实状态。

## 文档和注释验收

检查禁止句式和异常占位时，不要把禁用词直接写入长期文档。可以在本地临时拼接模式后执行：

```powershell
$patterns = @('这不' + '是', '不' + '是.*而' + '是', '不.{0,30}而' + '是', 'X' + 'X' + 'X')
rg -n ($patterns -join '|') README.md docs apps/workbench autoform_agent tests handoff docs/maintainer_onboarding
```

检查 Markdown 是否包含空链接、过期路径或没有依据的结论。新增结论要能回到源码、测试、命令输出、开发 DOCX 或用户明确要求。

## DOCX 输出验收

如果本轮创建或修改 DOCX：

1. 写入前确认目标 DOCX 时间戳。
2. 用 `python-docx` 抽取段落、表格和全文。
3. 检查禁用句式、异常占位、空来源和关键内容。
4. 检查字体 XML 绑定。
5. 若有可用 DOCX 转 PDF 环境，再做页面渲染抽查。
6. 若没有渲染环境，明确记录剩余版式风险。
