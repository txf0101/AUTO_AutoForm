# 05 开发修改作业指导书

本文件告诉维护者如何改项目。每次改动都要留下依据、测试和复盘。

## 第一步：判断改动类型

先把需求归到下面某一类：

| 类型 | 常见例子 | 先看文件 |
| --- | --- | --- |
| 启动与环境 | 启动器、端口、Python 环境、服务重启 | `start_autoform_agent.ps1`、`README.md`、`docs/beginner_onboarding_zh.md` |
| 前端界面 | Agent 图谱、按钮、状态面板、API key 面板 | `frontend/index.html`、`frontend/app.js`、`frontend/styles.css`、`frontend/README.md` |
| HTTP runtime | `/api/agent`、provider 配置、API key 边界 | `autoform_agent/http_bridge.py`、`autoform_agent/agent_runtime.py` |
| 多 Agent | 角色、路由、事件、中心计划、工具权限 | `autoform_agent/agent_system/`、`docs/multi_agent_architecture.md` |
| MCP 工具 | 外部 MCP host 工具、`autoform_` 工具注册 | `autoform_agent/mcp_server.py`、`autoform_agent/mcp_tools/` |
| 工程运行 | 示例工程解析、复制、打开窗口、求解 | `autoform_agent/project_workflow.py`、`solver.py`、`process.py` |
| 后处理 | 结果工程、截图、视角、动画、审阅计划 | `result_viewer.py`、`results.py`、`gui_automation.py` |
| 报告交付 | 报告草案、证据包、发布检查 | `report.py`、`release.py`、`results.py` |
| 文档 | README、新手文档、维护文档、复盘 | `README.md`、`docs/`、`handoff/`、`维护者入门阅读文档/` |

## 第二步：读对应开发文档

任何涉及需求、架构、代码、测试、README、入门文档、复盘或交付物的开发，都要先读 `VC开发文档\Auto_Autoform思路整理` 下的相关 DOCX。交付说明中要写明：

- 已读文件路径。
- 文件时间戳。
- 采用结论。
- 仍需验证的问题。

常用读取组合：

- 总体架构：`01_项目总览与系统架构.docx`。
- 中心 Agent 和工具治理：`02_项目中心Agent详细架构计划与任务目标.docx`。
- 上下文和补丁：`02_上下文信息结构体详细架构计划与任务目标.docx`。
- RAG 和证据：`02_RAG工艺数据库详细架构计划与任务目标.docx`。
- 需求、几何、材料、工艺：对应 `03_*.docx`。
- 脚本风险和 GUI：`04_柔性脚本L0至L4详细架构计划与任务目标.docx`。
- 前端图谱和状态：`05_AutoForm多Agent软件界面开发说明.docx`。
- 九个业务 Agent：`06_Agent开发规划_*.docx`。

## 第三步：找真正的责任层

不要直接在页面上修结果。按这个顺序定位：

1. 字段显示错：先看后端事件和响应，再看前端渲染。
2. Agent 选错：看 `orchestrator.py` 和 `registry.py`。
3. 工具没调用：看 `agent_runtime.py` 是否生成工具请求，再看 `tool_gateway.py` 是否拦截。
4. MCP host 工具不可用：看 `mcp_tools/` wrapper 和 `mcp_server.py` 注册。
5. 示例工程找不到：看 `paths.py` 和 `inventory.py`。
6. 复制或打开窗口异常：看 `project_workflow.py` 和 `process.py`。
7. 求解结果异常：看 `solver.py`、`results.py` 和 `output/project_runs`。

## 第四步：写代码时的注释规则

注释服务维护，不服务装饰。应该注释：

- 模块承担的职责。
- 函数输入、输出和安全边界。
- 为什么这里要拦截危险动作。
- 为什么这里要保留旧 role_id 或旧字段。
- 为什么这里要先复制工程再打开或求解。
- 为什么这里要隐藏明文 key。

不建议注释：

- 已经一眼能看懂的赋值。
- 与代码不一致的愿望。
- 空泛口号。
- 过期实现计划。

## 第五步：补测试

按修改范围选择测试：

| 修改范围 | 建议测试 |
| --- | --- |
| Agent runtime | `tests/test_agent_runtime.py` |
| 多 Agent 注册、路由、网关 | `tests/test_agent_system.py`、`tests/test_agent_system_runtime.py` |
| 工程复制、打开、求解计划 | `tests/test_project_workflow.py` |
| 启动器 | `tests/test_launcher_scripts.py` |
| 前端 DOM 和图谱 | `frontend/tests/smoke_test.py`、`frontend/tests/smoke-test.mjs` |
| 结果审阅 | `tests/test_result_viewer.py`、`tests/test_results.py` |
| MCP wrapper | `tests/test_mcp_tools.py` |

常用组合：

```powershell
& 'C:\Users\Tang Xufeng\.conda\envs\afagent\python.exe' -m pytest -q --basetemp=tmp\pytest_direct tests\test_agent_system.py tests\test_agent_system_runtime.py frontend\tests\smoke_test.py tests\test_agent_runtime.py tests\test_project_workflow.py tests\test_launcher_scripts.py
```

如果 pytest 报临时目录权限问题，使用 `--basetemp=tmp\pytest_direct`。

## 第六步：补文档

这些改动需要同步文档：

- 改启动命令、环境、端口：更新 `README.md`、`docs/beginner_onboarding_zh.md` 和本文件夹。
- 改前端页面、图谱或 API 面板：更新 `frontend/README.md` 和 `docs/api_runtime_call_chain.md`。
- 改 Agent 角色或路由：更新 `docs/multi_agent_architecture.md`、本文件夹和相关 DOCX 复盘说明。
- 改 MCP 工具：更新 `README.md`、`docs/beginner_onboarding_zh.md`、`codex_mcp_config.autoform-agent.toml` 的说明。
- 改工程运行或求解：更新 `README.md`、`docs/api_runtime_call_chain.md`、`docs/beginner_onboarding_zh.md`。

## 第七步：写复盘

每轮开发结束后，在 `handoff/` 新增一份 Markdown。建议包含：

- 用户请求。
- 已读资料路径、时间戳、采用结论。
- 旧问题判断。
- 本轮改动。
- 验证命令和结果。
- 仍需验证的问题。
- 方法沉淀。

复盘要回答价值问题：这个改动让项目更可维护、更可复用、更可验证的地方在哪里。

## 常见修改样板

### 新增一个 AutoForm 工具

1. 在业务模块写可测试函数。
2. 给函数补 docstring 和安全边界说明。
3. 在 `mcp_tools/` 对应模块加 wrapper。
4. 在 `register_all_tools()` 路径确认注册。
5. 如果 Agent 要调用，在 `tool_gateway.py` 加 `GatewayToolSpec`。
6. 如果 provider 工具意图要调用，在 `agent_runtime.py` 的 `_runtime_tool_registry()` 加白名单项。
7. 补测试和文档。

### 改前端 Agent 图谱

1. 先确认九个业务 Agent 名称有没有变化。
2. 改 `DEFAULT_AGENTS`、`AGENT_LABELS` 和 `AGENT_NODE_ALIASES`。
3. 改 `normalizeAgentState()`，保证完成态回到待命。
4. 改 CSS 时保持三列九宫格。
5. 补前端 smoke 测试。
6. 用页面回放确认运行态和完成态。

### 改工程复制或打开窗口

1. 先读 `project_workflow.py`。
2. 明确 `copy_project`、`open_gui`、`execute` 三个参数各自含义。
3. 没有批准时，网关应阻断受控参数。
4. 批准后应优先操作复制后的工程。
5. 不要让打开窗口自动触发求解。
6. 补 `tests/test_project_workflow.py` 和 `tests/test_agent_runtime.py`。
